# Vision Capital AI 学习笔记

## 为什么这个项目用 Python

Python 在 AI、数据处理、后端接口和任务调度领域都有成熟生态。这个项目既要做 Web API，也要做文档解析、向量化、RAG 和异步任务，Python 可以把这些部分较自然地串起来。

## FastAPI 在项目中的作用

FastAPI 是后端对外提供 REST API 的框架。注册、登录、项目管理、文件上传、AI 问答、报告生成等能力，都是通过 FastAPI 暴露给前端页面和 Swagger 的。

## Pydantic 是什么

Pydantic 主要负责数据校验和结构化。当前端传入 JSON 时，Pydantic 会检查字段是否齐全、类型是否正确，并把原始输入转换成更安全、更清晰的 Python 对象。

## SQLAlchemy 是什么

SQLAlchemy 是 Python 常用 ORM。它把数据库表映射成 Python 类，让我们用对象和查询表达式来操作 PostgreSQL，而不是把所有逻辑都写成原始 SQL。

## Alembic 是什么

Alembic 是 SQLAlchemy 常用的数据库迁移工具。它帮助团队记录数据库结构变化，例如新增表、修改字段、增加索引，这样不同环境都能同步更新。

## Celery + Redis 为什么用于异步任务

文件上传后的解析、切片、向量化可能比较慢，如果这些操作都阻塞在上传请求里，用户体验会很差。Celery 负责执行后台任务，Redis 充当消息代理，这样上传接口可以先返回，后续再异步完成解析。

## R2 为什么可以用 boto3 操作

Cloudflare R2 提供的是 S3 compatible API。`boto3` 本来是 AWS S3 的 Python SDK，但因为接口兼容，所以只需要替换 endpoint 和凭证，就可以用类似方式访问 R2。

## 什么是 Embedding

Embedding 是把文本变成向量的过程。变成向量后，系统就能通过数学方式比较语义相似度，从而找到和用户问题最相关的文档片段。

## 什么是 pgvector

pgvector 是 PostgreSQL 的向量扩展。它让数据库可以直接存储向量，并执行相似度检索，这样 RAG 的知识库不一定要额外引入单独的向量数据库。

## 什么是 RAG

RAG 全称是 Retrieval-Augmented Generation，意思是“检索增强生成”。系统先从知识库里找相关资料，再把资料带给大模型生成回答，因此回答更贴近项目文件，而不只是模型自己的通用知识。

## 什么是 Prompt

Prompt 可以理解成给大模型的任务说明书。好的 Prompt 会明确角色、任务目标、上下文和输出要求，从而让生成结果更稳定、更可控。

## 什么是 AI Agent

AI Agent 通常不只是“一次问答”，而是能分步骤思考、调用工具、规划任务、观察结果再继续执行的系统。当前版本先实现 RAG 问答和报告生成，不做复杂 Agent，但后续可以扩展成自动尽调、自动追踪舆情、自动输出投后提醒的多工具智能体。

## 投前、投中、投后在系统中的业务含义

- 投前：重点关注行业、团队、市场空间、商业模式、风险点和尽调问题。
- 投中：重点关注交易推进进度、核心条款、财务验证和舆情变化。
- 投后：重点关注经营数据、里程碑达成情况、风险预警和赋能建议。
